    • Exercise #5, Is it possible to rewrite the previous example to be linear?

    I do not think it is possible for an arithmatic operation to be linear when trying to sort a list to find the n'th largest number. I'm certain that an algorithm that goes beyond trying to find the smallest or largest number will always end up being 0(n^2) or O(n log n). An algorithm that needs to compare numbers against every other number in a list cannot be linear in its timescale. This stands when n is any value, but as stated, I believe it is possible for it to be linear when n is looking for the smallest or largest number. It only needs to make one trip through the list when finding either the smallest or largest, picking up whichever it needs and dropping the previous value.
